# Kubernetes Reliability Lab

Production-style Kubernetes reliability engineering environment focused on:

- Observability
- Incident response
- CrashLoopBackOff troubleshooting
- Resource exhaustion testing
- MTTR reduction
- Kubernetes monitoring

## Stack
- Kubernetes
- Prometheus
- Grafana
- Loki
- Helm
- Terraform

## Features
- Pod failure simulations
- Node pressure testing
- Alerting pipelines
- Dashboarding
- SLO monitoring
- Failure analysis workflows

## Goals
- Improve reliability engineering practices
- Build operational visibility
- Reduce mean time to recovery (MTTR)
- Simulate production-style failures
